## Status: done (2026-07-10)

live server to client-side markdown rendering. Investigation showed the
server already renders fresh per request; the only real cache vector is
browser HTTP caching (no cache headers sent). Decision: add
`Cache-Control: no-store` to dynamic responses instead of re-architecting.
