# OMD-019 — Marketplace SEO: searchable displayName + expanded keywords

**Status:** done · **Date:** 2026-07-20 · Follows [OMD-016](OMD-016.md) (listing overhaul).

## Why
Growth question from the owner ("ทำอย่างไรให้ extension มีคนใช้เยอะๆ"). The
biggest cheap lever for a VS Code extension is marketplace discoverability.
Two gaps remained after OMD-016:

- **`displayName` was the bare brand `"OpenMD"`** — carries zero search
  weight even though VS Code ranks displayName heavily. A user searching
  "markdown preview browser" never matched on the name.
- **Keywords omitted long-tail terms OpenMD actually wins on.** The category
  "markdown preview" is saturated (built-in + Markdown Preview Enhanced +
  Markdown All in One). Strategy: don't fight the head term — rank for
  long-tail where OpenMD is genuinely the best answer.

## Changes (package.json, metadata only)
- `displayName`: `"OpenMD"` → `"OpenMD — Markdown Preview in Browser"`.
  Keeps "OpenMD" as the lead brand word; adds the "Markdown Preview" search
  term + the "Browser" differentiator. Follows the GitLens/Prettier
  "Name — tagline" convention; descriptive, not keyword stuffing.
- Keywords 15 → 23. Added: `pdf`, `print` (browser Ctrl+P → save-as-PDF is a
  real OpenMD path, unlike the webview-only alternatives), `readme`,
  `documentation`, `technical writing`, `cursor`, `windsurf`, `vscodium`
  (Open VSX reach — already advertised in the README). Dropped `export` on
  purpose: no export *button* exists, would overpromise.

## Verification
- `node -e` dump: displayName correct, 23 keywords (≤ 30 cap).
- `npx vsce ls` — manifest accepted.
- `npm run test:unit` — 60 passing.
