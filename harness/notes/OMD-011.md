# OMD-011: CI release pipeline (GitHub Actions publish on tag)

## Status: in_progress (2026-07-10)

## Context
Manual publishing required pasting PATs into chat (scrubbed + rotated each
time). User asked to store keys in GitHub instead. Repo is public — safe:
Actions Secrets are write-only/masked, fork PRs never receive secrets, and
the tag-push trigger is limited to users with push access.

## Design
- `.github/workflows/release.yml`: on `v*` tag → npm ci → unit tests →
  vsce package → vsce publish (VSCE_PAT) → ovsx publish (OVSX_TOKEN) →
  attach vsix to the GitHub Release.
- Actions pinned to commit SHAs (checkout@34e11487..., setup-node@49933ea...).
- Integration tests are host-only (need a live VS Code); CI gates on unit tests.
- Token issuance guide lives in docs/RELEASE.md so the human never has to
  remember the Azure DevOps / Open VSX steps again.
