# OMD-021 — Codex subagent roles (.codex/agents) for delegation

Status: done (2026-07-20)

## What

Ported the project-level Codex CLI subagent setup from AuttapOnG/at-tools-web
(private repo, fetched via gh), adapted to OpenMD:

- `.codex/config.toml` — `[agents] max_threads = 3, max_depth = 1`
- `.codex/agents/implementation-worker.toml` — bounded implementation tasks
  (model gpt-5.6-terra, effort medium, sandbox workspace-write)
- `.codex/agents/test-reviewer.toml` — verification/check runs (effort low)

Adaptations vs the at-tools-web originals: Next.js/Supabase constraints
replaced with OpenMD ones — narrow checks are `npm run test:unit` /
`npm run compile`; preview-server integration tests are host-only (the
sandbox cannot bind ports); protected paths are AGENTS.md, CLAUDE.md,
docs/, harness/, .codex/, package-lock.json, .github/workflows/, and
package.json release metadata.

## How Codex loads them (verified against source + live probes)

- Codex CLI's config layer stack includes the repo layer
  `$(git toplevel)/.codex/config.toml`, enabled only when the project is
  trusted (`[projects."…/Project/OpenMD"] trust_level = "trusted"` in
  `~/.codex/config.toml` — already set on this machine). Each layer also
  auto-discovers `<layer folder>/agents/*.toml` as agent roles
  (openai/codex: codex-rs/core/src/config/agent_roles.rs,
  codex-rs/config/src/loader/mod.rs).
- `multi_agent` feature: stable + enabled here (codex-cli 0.144.2).
- Works identically under `codex exec` (non-interactive): the primary agent
  gets spawn_agent / wait / close_agent tools. Who typed the prompt is
  irrelevant, so Claude-dispatched runs (cli-subagents skill) can delegate.

## Gotchas (live-tested 2026-07-20)

- `spawn_agent` with `agent_type` REQUIRES `fork_turns=none` (or a partial
  fork). With the default full-history fork it errors: "Full-history forked
  agents inherit the parent agent type, model, and reasoning effort".
- Unknown roles fail with `unknown agent_type '…'` — a successful spawn is
  therefore real proof the role loaded.
- Do NOT trust the model's own claim about which roles exist: asked directly
  it answered "none" while both roles actually worked. Verify by spawning.
- Verification probes (codex exec, read-only sandbox): test-reviewer →
  `/root/probe_2` answered "ok"; implementation-worker → `/root/probe_3`
  answered "ok".

## Using it in delegation (cli-subagents workflow)

The dispatch prompt can now say e.g. "delegate bounded implementation steps
to implementation-worker and run checks via test-reviewer (spawn with
fork_turns=none)". Subagents inherit the session sandbox (no network, no
.git writes, no port binding), so plan Global Constraints still hold, and
all threads report into the same session output so the watchdog keeps
working. Subagents also consume the owner's OpenAI quota — up to 3
concurrent threads.
