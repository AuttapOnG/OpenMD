# OMD-008 — Bundle with esbuild + full offline support

**Status:** done

Acceptance criteria: harness/feature_list.json (OMD-008). Spec: docs/specs/OMD-008-bundle-offline.md.

## Notes

(Record decisions, surprises, and gotchas here while implementing.)

- Design decision (2026-07-08): shared-assets approach — assets copied to
  extension media/ at build time and to .temp/assets/ on activate; rejected
  inlining (3 MB per HTML) and CDN-with-fallback (still online-dependent).
- Browser HTML must use absolute file:// asset paths — mirror paths have
  varying depth, so relative paths would break.
- 2026-07-08: Implementation started on branch feat/omd-008-bundle-offline.
- Task 1: esbuild bundling added. `markdown-it-emoji` v3 has no default ESM export, so the build uses an esbuild-only shim mapping the existing default import to the package's `full` export until render extraction makes the source explicit.
- Task 2: extracted `src/render.ts` and unit tests. TypeScript expects a default import for `markdown-it-emoji`, while runtime exposes `full`, so `render.ts` resolves either shape.
- Task 3: `extension.ts` now stages offline assets after global temp cleanup and uses `generateHtml` for both browser and webview rendering. Manual F5 smoke test is intentionally left for the human per task instructions.
- Task 4: packaged `openmd-1.1.0.vsix`; final size is 924K (`vsce` reported 924.33 KB, 12 files). Contents include `dist/extension.js` and `media/{mermaid.min.js,highlight.min.js,github.min.css,github-dark.min.css}` with zero `node_modules` entries. Manual offline install test intentionally left for the human per task instructions.
