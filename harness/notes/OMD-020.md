# OMD-020 — Export to self-contained HTML + Export to PDF

**Status:** done · **Date:** 2026-07-20

Plan: [docs/plans/2026-07-20-omd-020-export-html-pdf.md](../../docs/plans/2026-07-20-omd-020-export-html-pdf.md)

## What shipped
Two right-click commands on `.md` files (also in the editor menu + command palette):
- **Export to HTML** — writes a single self-contained `<name>.html` next to the
  source (Reveal/Open actions), meant to be sent to someone with no VS Code / no
  internet.
- **Export to PDF** — opens the doc in the browser with `?print=1`; the page
  auto-opens the print dialog after rendering, user picks "Save as PDF". No PDF
  engine bundled.

## How it works
- `src/render.ts`: refactored `htmlTemplate` to take an `AssetTags` object, so it
  emits either *linked* tags (`generateHtml`, unchanged behavior) or *inlined*
  tags (`generateStandaloneHtml`). Conditional inlining: Mermaid JS (2.6 MB) only
  when the body has `class="mermaid"`; KaTeX css/fonts only when it has
  `class="katex"`. No live-reload script in standalone output.
- `embedKatexFonts(cssText, readFont)` (pure, injected reader) rewrites
  `url(fonts/*.woff2)` to base64 `data:` URIs and drops the `.woff`/`.ttf`
  fallbacks (only woff2 is vendored).
- Shared template gained an `@media print` block (hides toggle/copy buttons,
  page-break-avoid on pre/table/mermaid/katex) and a client-side
  `?print=1` → `window.print()` trigger. Mermaid init/run is now guarded with
  `typeof mermaid !== 'undefined'` so a diagram-less standalone file doesn't crash.
- `src/extension.ts`: `buildInlineAssets()` reads the `media/` files; two commands
  wired up; PDF reuses the preview server (`server.url(urlPath) + '?print=1'`) with
  a standalone-file fallback when the server is unavailable.

  2 katex-embed). Includes server tests the sandbox can't run.
- Real-asset check (scratchpad script): sample with mermaid+math+code = 3.16 MB
  (mermaid inlined); **plain doc = 176.6 KB** (no mermaid, no katex — conditional
  inlining confirmed). 20 woff2 fonts embedded as data URIs; no `/assets/`, no
  `file:`, no relative `url(fonts/)` in output.

  network request is the HTML itself (200) — no external fetch. Mermaid → 1 SVG,
  math → 2 `.katex`, code highlighted, task checkboxes + theme toggle present.
  `?print=1` opened the native print dialog (Destination: Save as PDF) with
  `@media print` applied (toggle/copy hidden) — verified visually.
